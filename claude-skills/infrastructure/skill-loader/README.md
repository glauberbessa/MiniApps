# /skill-loader

**Discover and understand which skill to use for your specific need.**

Use this skill when you're unsure what to do or need to find the right tool for your task.

---

## What This Skill Does

- 🎯 **Finds relevant skills** for your problem
- 📖 **Explains when to use** each skill
- 🗺️ **Shows skill relationships** (which skills work together)
- 💡 **Suggests workflows** (sequence of skills)
- 🚀 **Recommends next steps** (what to do after)

---

## When to Use

✅ "I need to plan a feature"
✅ "How do I debug this issue?"
✅ "What's the best way to structure an API?"
✅ "I want to create a new project"
✅ "My code needs review"
❌ Direct implementation (use domain-specialist skill)
❌ Quick syntax help (use specific skill)

---

## How It Works

1. Describe your task/problem
2. Skill suggests 2-3 relevant skills
3. Shows when to use each
4. Recommends which to start with
5. Provides next steps

## Common Workflows

### "I want to build a React component"
```
1. /skill-loader → Find relevant skill
   ↓
2. /react-expert → Implementation guidance
   ↓
3. /code-reviewer → Validate
```

### "I found a bug in my API"
```
1. /debugging-techniques → Diagnostic approach
   ↓
2. /api-architect OR /backend-expert → Fix guidance
   ↓
3. /code-reviewer → Validate fix
```

### "Starting a new project"
```
1. /skill-loader → Find template match
   ↓
2. /scaffold-[template] → Generate
   ↓
3. /[domain-specialist] → Customize
   ↓
4. /code-reviewer → Final checks
```

### "Code needs optimization"
```
1. /skill-loader → Find optimization expert
   ↓
2. /performance-profiler → Analyze
   ↓
3. /[specialist] → Implement fixes
   ↓
4. /code-reviewer → Verify improvements
```

## Tips for Using Skills

1. **Start with goal** - What are you trying to accomplish?
2. **Find expert** - Use `/skill-loader` if unsure
3. **Get plan** - Use `/plan-writer` for complex work
4. **Execute** - Use domain-specific skill
5. **Validate** - Use `/code-reviewer` before shipping
